# 代码安全建设

- 背景
  - 勒索病毒暴露出相关人员的安全意识薄弱，同时研发部的安全也要引起重视。本次会议主要讨论代码层面要关注哪些点
- 最少暴露原则
  - 不该给前端的数据不应该暴露
    - 例子 1：国际机票订单详情显示到了前端
    - 例子 2：基础数据保险 保险成本暴露给了前端
  - 不该暴露的 API
  - 基础数据接口是否要拿到 token 才能访问
  - gateway 配置
  - 密码明文传递 不是问题 https 解决
- 关键数据以服务端为准，三方数据都要做检查
  - 金钱有关信息要做校验
  - 计算不依靠前端
  - 供应商数据检查
- 服务端安全
  - 接口
  - 鉴权，防止无权限的客户访问接口
  - API 防刷
  - 数据访问权限
  - 自己能访问自己数据
  - 管理员能访问下级数据
  - 同级之间不可互访问信息
  - SQL 注入防止
